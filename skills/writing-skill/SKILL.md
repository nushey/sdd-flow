# Writing Skill: Structured Planning & Documentation

This skill provides a framework for writing high-quality technical documents and implementation plans within the SDD flow.

## Guidelines

1.  **Objective First**: Always start with a clear, one-sentence objective of what is being planned or documented.
2.  **Hierarchical Structure**: Use Markdown headers (#, ##, ###) to create a clear information hierarchy.
3.  **Actionable Items**: When writing plans or tasks, use imperative verbs (e.g., "Implement", "Create", "Configure").
4.  **Verification Steps**: Every plan or task must include a "How to verify" section or specific acceptance criteria.
5.  **Context Preservation**: Reference existing files, patterns, and conventions explicitly.

## Structure for Plans

- **Goal**: What are we achieving?
- **Pre-requisites**: What needs to be in place?
- **Steps**: Atomic, sequential actions.
- **Verification**: How do we know it works?

## Structure for Design Docs

- **Technical Approach**: High-level logic.
- **Components Affected**: List of modules/files.
- **Data Flow**: How information moves.
- **Trade-offs**: Why this approach was chosen.
